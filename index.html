Décembre 2025

Classification de SMS Spam
Prétraitement Textuel & Apprentissage Profond

Projet de classification binaire (Ham vs Spam) avec un pipeline complet : nettoyage du texte, tokenisation, lemmatisation, vectorisation TF-IDF, réduction de dimension (autoencodeur) et embeddings BERT pour une compréhension sémantique.

  • Voir sur GitHub
Approches

Méthodologie
& Pipeline

Prétraitement textuel (minuscules, nettoyage, stopwords, lemmatisation), puis vectorisation TF-IDF. Comparaison de trois approches : TF-IDF + réseau dense, autoencodeur (réduction non-supervisée) et BERT (embeddings contextuels).

Dataset

SMS Spam Collection
Déséquilibre

Dataset de 5 572 SMS (≈86.6% Ham, ≈13.4% Spam). Le déséquilibre des classes peut biaiser un modèle vers la classe majoritaire, donc l’évaluation ne doit pas se limiter à l’accuracy (Recall et AUC-ROC sont essentiels).

Résultats

Comparaison des modèles
Performance

TF-IDF offre un excellent rapport performance / simplicité. BERT obtient la meilleure performance globale (AUC très élevée). L’autoencodeur échoue à détecter les spams (recall spam ≈ 0% malgré une accuracy trompeuse).
